## 0️⃣ 先修导读：服务治理到底在治什么？

> 单体项目不存在"服务治理"问题，因为整个系统就是一个服务。拆成微服务后，**服务多了，需要管理它们**，这就是服务治理。

---

### 🔍 1. 服务治理 = 管理微服务的"生命周期"

**想象一个外卖平台拆成多个微服务**：
- 用户服务（用户注册/登录）
- 商品服务（商品上架/下架）
- 订单服务（下单/取消）
- 支付服务（支付宝/微信）

**问题来了**：
1. 订单服务怎么知道用户服务在哪里？（服务发现）
2. 支付服务挂了，订单还在创建，怎么办？（熔断降级）
3. 某个用户疯狂下单，怎么限制？（限流）
4. 订单服务响应慢，导致整个系统卡死，怎么排查？（链路追踪）

**服务治理要解决的问题**：
- 🔍 **服务发现**：我需要调用"用户服务"，但它在哪台机器的哪个端口？
- ⚡ **熔断限流**：某个服务挂了，不能让整个系统崩溃
- 📊 **健康检查**：怎么知道某个服务还活着？
- 🔧 **链路追踪**：请求跨了 5 个服务，怎么知道慢在哪一步？

---

### 🔍 2. 熔断器：为什么需要"保险丝"？

#### ⚡ 生活类比：保险丝

电流过大 → 保险丝熔断 → 切断电路 → 保护电器不被烧坏 → 换保险丝 → 恢复

**微服务里的保险丝**：

某个服务（如商品服务）响应慢/挂了 → 熔断器打开 → 后续请求直接返回"服务不可用"（不继续调用已故障的服务） → 防止级联崩溃 → 等待一段时间 → 放行几个"探测请求" → 服务恢复正常 → 熔断器关闭

**三态转换**：

```
[CLOSED关闭] ---故障超过阈值--> [OPEN打开]
     ↑                              ↓
     ←←←←←← 探测成功 ←←←←←←←←←←←
              ↓
        [HALF-OPEN半开]
              ↓
         探测失败 → [OPEN]
         探测成功 → [CLOSED]
```

#### 熔断 vs 限流的区别

| 概念 | 熔断（Circuit Breaker）| 限流（Rate Limiting）|
|------|---------------------|--------------------|
| 触发条件 | 下游服务**故障**（超时/错误）| 请求量**过多** |
| 目的 | 防止**级联崩溃** | 保护**系统不被压垮** |
| 效果 | 后续请求直接失败，不调用故障服务 | 超出阈值的请求被拒绝 |

```typescript
// examples/stage7-service-governance/01-consul/consul.service.ts

import { Injectable, Logger, OnModuleInit } from '@nestjs/common';
import { ConfigService } from '@nestjs/config';
import { HttpService } from '@nestjs/axios';
import { firstValueFrom } from 'rxjs';

@Injectable()
export class ConsulService implements OnModuleInit {
  private readonly logger = new Logger(ConsulService.name);
  private readonly consulUrl: string;

  constructor(
    private readonly configService: ConfigService,
    private readonly httpService: HttpService,
  ) {
    this.consulUrl = this.configService.get('CONSUL_URL', 'http://localhost:8500');
  }

  // ========== 服务注册 ==========

  async registerService(service: {
    name: string;
    id: string;
    address: string;
    port: number;
    healthCheck?: { path: string; interval: string };
  }): Promise<void> {
    const payload = {
      Name: service.name,
      ID: service.id,
      Address: service.address,
      Port: service.port,
      Tags: ['nestjs', 'api'],
      Check: service.healthCheck
        ? {
            HTTP: `${this.consulUrl}/api/health`,
            Interval: service.healthCheck.interval,
            Timeout: '5s',
            DeregisterCriticalServiceAfter: '30s',
          }
        : undefined,
    };

    await firstValueFrom(
      this.httpService.put(`${this.consulUrl}/v1/agent/service/register`, payload),
    );

    this.logger.log(`服务已注册: ${service.name}:${service.id}`);
  }

  // ========== 服务注销 ==========

  async deregisterService(serviceId: string): Promise<void> {
    await firstValueFrom(
      this.httpService.put(`${this.consulUrl}/v1/agent/service/deregister/${serviceId}`),
    );
    this.logger.log(`服务已注销: ${serviceId}`);
  }

  // ========== 服务发现 ==========

  async discoverService(serviceName: string): Promise<{ address: string; port: number }[]> {
    const response = await firstValueFrom(
      this.httpService.get(`${this.consulUrl}/v1/health/service/${serviceName}`, {
        params: { passing: true }, // 只返回健康的实例
      }),
    );

    const services = response.data as any[];
    return services.map((s) => ({
      address: s.Service.Address,
      port: s.Service.Port,
    }));
  }

  // ========== KV 存储 ==========

  async getKv(key: string): Promise<string | null> {
    const response = await firstValueFrom(
      this.httpService.get(`${this.consulUrl}/v1/kv/${key}`),
    );
    const data = response.data as any[];
    if (!data || data.length === 0) return null;
    return Buffer.from(data[0].Value, 'base64').toString('utf8');
  }

  async setKv(key: string, value: string, opts?: { ttl?: number }): Promise<void> {
    const fullKey = opts?.ttl ? `${key}?ttl=${opts.ttl}` : key;
    await firstValueFrom(
      this.httpService.put(`${this.consulUrl}/v1/kv/${fullKey}`, value),
    );
  }

  async deleteKv(key: string): Promise<void> {
    await firstValueFrom(
      this.httpService.delete(`${this.consulUrl}/v1/kv/${key}`),
    );
  }

  // ========== 监听配置变更 ==========

  async watchKv(key: string, callback: (value: string) => void): Promise<void> {
    // Consul 的 Long Polling 机制
    const poll = async () => {
      try {
        const response = await firstValueFrom(
          this.httpService.get(`${this.consulUrl}/v1/kv/${key}`, {
            params: { wait: '60s', index: (response.data as any)?.[0]?.ModifyIndex },
          }),
        );
        const data = response.data as any[];
        if (data && data.length > 0) {
          const value = Buffer.from(data[0].Value, 'base64').toString('utf8');
          callback(value);
        }
      } catch (e) {
        this.logger.error('配置监听错误:', e);
      }
      poll(); // 继续轮询
    };
    poll();
  }

  onModuleInit() {
    this.logger.log(`Consul 服务已初始化: ${this.consulUrl}`);
  }
}
```

## 3️⃣ 熔断器（Circuit Breaker）

### 3.1 熔断器原理

```
┌─────────────────────────────────────────────────────────────┐
│                    熔断器状态机                               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   熔断器初始状态：CLOSED（关闭）                            │
│                                                             │
│   CLOSED（关闭）→ 正常调用 → 成功计数 +1                    │
│        ↓ 失败次数超过阈值                                   │
│   OPEN（打开）→ 拒绝所有请求 → 返回降级响应                 │
│        ↓ 等待一段时间后                                     │
│   HALF_OPEN（半开）→ 允许一个请求尝试                       │
│        ↓ 成功 → CLOSED / 失败 → OPEN                       │
│                                                             │
│   状态转移条件：                                            │
│   CLOSED → OPEN：连续失败次数 > 失败阈值（默认 5 次）       │
│   OPEN → HALF_OPEN：熔断等待时间到期（默认 30 秒）          │
│   HALF_OPEN → CLOSED：探测请求成功                          │
│   HALF_OPEN → OPEN：探测请求失败                            │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 熔断器实现

```typescript
// examples/stage7-service-governance/02-circuit-breaker/circuit-breaker.ts

import { Injectable, Logger } from '@nestjs/common';

export enum CircuitBreakerState {
  CLOSED = 'CLOSED',
  OPEN = 'OPEN',
  HALF_OPEN = 'HALF_OPEN',
}

export interface CircuitBreakerOptions {
  name: string;
  failureThreshold: number;  // 失败阈值（默认 5）
  successThreshold: number;  // 成功阈值（默认 2）
  timeout: number;           // 熔断等待时间（默认 30000ms）
}

@Injectable()
export class CircuitBreakerService {
  private readonly logger = new Logger(CircuitBreakerService.name);
  private state: CircuitBreakerState = CircuitBreakerState.CLOSED;
  private failureCount = 0;
  private successCount = 0;
  private nextAttempt: number;

  constructor(private readonly options: Partial<CircuitBreakerOptions> = {}) {
    const opts: Required<CircuitBreakerOptions> = {
      name: 'default',
      failureThreshold: 5,
      successThreshold: 2,
      timeout: 30000,
      ...options,
    };
    this.options = opts;
  }

  async execute<T>(fn: () => Promise<T>, fallback?: () => Promise<T>): Promise<T> {
    const { name, failureThreshold, successThreshold, timeout } = this.options as Required<CircuitBreakerOptions>;

    // 检查熔断状态
    if (this.state === CircuitBreakerState.OPEN) {
      if (Date.now() < this.nextAttempt) {
        this.logger.warn(`CircuitBreaker [${name}] OPEN，拒绝请求`);
        if (fallback) return fallback();
        throw new Error(`CircuitBreaker [${name}] OPEN`);
      }
      // 超时，转为 HALF_OPEN
      this.state = CircuitBreakerState.HALF_OPEN;
      this.logger.log(`CircuitBreaker [${name}] OPEN → HALF_OPEN`);
    }

    try {
      const result = await fn();

      // 成功
      this.onSuccess();

      return result;
    } catch (error) {
      // 失败
      this.onFailure();

      if (fallback) return fallback();

      throw error;
    }
  }

  private onSuccess() {
    const { name, successThreshold } = this.options as Required<CircuitBreakerOptions>;

    if (this.state === CircuitBreakerState.HALF_OPEN) {
      this.successCount++;
      if (this.successCount >= successThreshold) {
        this.state = CircuitBreakerState.CLOSED;
        this.failureCount = 0;
        this.successCount = 0;
        this.logger.log(`CircuitBreaker [${name}] HALF_OPEN → CLOSED`);
      }
    } else {
      this.failureCount = 0;
    }
  }

  private onFailure() {
    const { name, failureThreshold, timeout } = this.options as Required<CircuitBreakerOptions>;

    this.failureCount++;

    if (this.state === CircuitBreakerState.HALF_OPEN) {
      // HALF_OPEN 状态下再失败，立即 OPEN
      this.state = CircuitBreakerState.OPEN;
      this.nextAttempt = Date.now() + timeout;
      this.logger.warn(`CircuitBreaker [${name}] HALF_OPEN → OPEN`);
    } else if (this.failureCount >= failureThreshold) {
      this.state = CircuitBreakerState.OPEN;
      this.nextAttempt = Date.now() + timeout;
      this.logger.warn(`CircuitBreaker [${name}] CLOSED → OPEN (failureCount: ${this.failureCount})`);
    }
  }

  getState(): CircuitBreakerState {
    return this.state;
  }

  reset() {
    this.state = CircuitBreakerState.CLOSED;
    this.failureCount = 0;
    this.successCount = 0;
  }
}

// ========== NestJS Guard 集成 ==========

@Injectable()
export class CircuitBreakerGuard implements CanActivate {
  private readonly breaker: CircuitBreakerService;

  constructor(
    private readonly options: CircuitBreakerOptions,
    private readonly httpService: HttpService,
  ) {
    this.breaker = new CircuitBreakerService(options);
  }

  async canActivate(context: ExecutionContext): Promise<boolean> {
    const fallback = () => {
      // 降级响应
      const response = context.switchToHttp().getResponse();
      response.status(503).json({
        statusCode: 503,
        message: '服务暂时不可用，请稍后重试',
        error: 'Service Unavailable',
      });
      return false;
    };

    try {
      const result = await this.breaker.execute(
        () => this.httpService.get('http://external-api/health').toPromise(),
        fallback,
      );
      return true;
    } catch {
      return false;
    }
  }
}
```

## 4️⃣ 限流（Rate Limiter）

### 4.1 限流算法

```
┌─────────────────────────────────────────────────────────────┐
│                    限流算法对比                               │
├────────────────┬────────────────┬────────────────────────────┤
│   令牌桶算法    │   漏桶算法      │   滑动窗口算法              │
├────────────────┼────────────────┼────────────────────────────┤
│ 以固定速率生成令牌│  水以固定速率    │  时间窗口内统计请求次数     │
│ 请求消耗令牌     │  从桶底漏出     │  超出则拒绝               │
│ 有令牌则通过     │  超桶则拒绝     │                            │
│ 允许突发流量     │  流量平滑       │  精度高，内存占用大         │
│                │                │                            │
│ Redis: SETNX  │  Redis: LIST   │  Redis: ZSET (时间戳为分)  │
└────────────────┴────────────────┴────────────────────────┘
```

### 4.2 NestJS Throttler（@nestjs/throttler）

```typescript
// examples/stage7-service-governance/02-circuit-breaker/rate-limiter.ts

import { Module } from '@nestjs/common';
import { ThrottlerModule, ThrottlerGuard } from '@nestjs/throttler';
import { APP_GUARD } from '@nestjs/core';

@Module({
  imports: [
    ThrottlerModule.forRoot([
      {
        name: 'short',           // 限制器名称
        ttl: 60000,             // 时间窗口（毫秒）
        limit: 10,              // 窗口内最多 10 次请求
      },
      {
        name: 'long',
        ttl: 600000,           // 10 分钟窗口
        limit: 100,
      },
      {
        name: 'strict',
        ttl: 1000,            // 1 秒窗口
        limit: 3,             // 每秒最多 3 次
      },
    ]),
  ],
  providers: [
    {
      provide: APP_GUARD,
      useClass: ThrottlerGuard,
    },
  ],
})
export class AppModule {}

// ========== 控制器级别限流 ==========

@Controller('api')
export class ApiController {
  // 默认使用全局限流（short）
  @Get('standard')
  standardEndpoint() {
    return '标准限流';
  }

  // 指定限流器
  @Throttle({ strict: { limit: 5, ttl: 1000 } })
  @Get('strict')
  strictEndpoint() {
    return '每秒最多5次';
  }

  // 跳过限流（如健康检查）
  @SkipThrottle()
  @Get('health')
  healthEndpoint() {
    return { status: 'ok' };
  }

  // 手动设置
  @Throttle({ long: { limit: 20, ttl: 600000 } })
  @Get('many')
  manyEndpoint() {
    return '放宽限制';
  }
}
```

## 5️⃣ 链路追踪（Distributed Tracing）

### 5.1 链路追踪原理

```
┌─────────────────────────────────────────────────────────────┐
│                    链路追踪原理                               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Trace: 一次完整的请求链路                                    │
│  Span: 链路中的一个操作单元                                  │
│                                                             │
│  Trace: [Span A] → [Span B] → [Span C] → [Span D]          │
│           ↓                                                 │
│        traceId: abc123                                     │
│        spans: A, B, C, D                                   │
│                                                             │
│  请求进入：生成 traceId + spanId                            │
│  跨服务调用：traceId 传递，spanId 新生成                     │
│  所有 Span 上报到追踪系统（Jaeger/Zipkin）                   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 5.2 OpenTelemetry 实现

```typescript
// examples/stage7-service-governance/03-tracing/tracing.service.ts

import { Injectable, OnModuleInit } from '@nestjs/common';
import { NodeSDK } from '@opentelemetry/sdk-node';
import { JaegerExporter } from '@opentelemetry/exporter-jaeger';
import { Resource } from '@opentelemetry/resources';
import { SemanticResourceAttributes } from '@opentelemetry/semantic-conventions';
import { trace, Span, SpanStatusCode, context } from '@opentelemetry/api';

@Injectable()
export class TracingService implements OnModuleInit {
  private sdk: NodeSDK;

  onModuleInit() {
    this.sdk = new NodeSDK({
      resource: new Resource({
        [SemanticResourceAttributes.SERVICE_NAME]: 'user-service',
        [SemanticResourceAttributes.SERVICE_VERSION]: '1.0.0',
      }),
      traceExporter: new JaegerExporter({
        endpoint: 'http://localhost:14268/api/traces',
      }),
    });

    this.sdk.start();
    console.log('OpenTelemetry 已初始化');
  }

  // ========== 创建 Span ==========

  async trace<T>(
    name: string,
    fn: (span: Span) => Promise<T>,
  ): Promise<T> {
    const tracer = trace.getTracer('user-service');

    return tracer.startActiveSpan(name, async (span) => {
      try {
        const result = await fn(span);
        span.setStatus({ code: SpanStatusCode.OK });
        return result;
      } catch (error) {
        span.setStatus({ code: SpanStatusCode.ERROR, message: error.message });
        span.recordException(error);
        throw error;
      } finally {
        span.end();
      }
    });
  }

  // ========== 跨服务传递 Context ==========

  injectTraceContext(headers: Record<string, string>) {
    const carrier: any = {};
    trace.getTracer('user-service').startActiveSpan('producer', (span) => {
      trace.getActiveSpan().inject(trace.ContextSpanKind.PRODUCER, carrier);
      span.end();
    });
    return carrier;
  }

  extractTraceContext(carrier: Record<string, string>) {
    return trace.getTracer('user-service').startActiveSpan('consumer', (span) => {
      // 使用 carrier 中的 trace context
      return span;
    });
  }
}
```

### 5.3 追踪拦截器

```typescript
// examples/stage7-service-governance/03-tracing/tracing.interceptor.ts

import {
  Injectable, NestInterceptor, ExecutionContext, CallHandler, Logger
} from '@nestjs/common';
import { Observable } from 'rxjs';
import { tap } from 'rxjs/operators';
import { trace, SpanStatusCode } from '@opentelemetry/api';

@Injectable()
export class TracingInterceptor implements NestInterceptor {
  private readonly logger = new Logger(TracingInterceptor.name);

  intercept(context: ExecutionContext, next: CallHandler): Observable<any> {
    const request = context.switchToHttp().getRequest();
    const { method, url } = request;
    const traceId = trace.getActiveSpan()?.spanContext().traceId || 'no-trace';

    const now = Date.now();

    return next.handle().pipe(
      tap({
        next: () => {
          const duration = Date.now() - now;
          this.logger.log(`[${traceId}] ${method} ${url} - ${duration}ms`);
        },
        error: (error) => {
          const duration = Date.now() - now;
          this.logger.error(`[${traceId}] ${method} ${url} - ${duration}ms - ${error.message}`);
        },
      }),
    );
  }
}

// 注册为全局拦截器
// app.module.ts
{
  provide: APP_INTERCEPTORS,
  useClass: TracingInterceptor,
}
```

### 6.1 @nestjs/terminus 健康检查

```typescript
// examples/stage7-service-governance/04-health-check/health.controller.ts

import { Controller, Get } from '@nestjs/common';
import {
  HealthCheck,
  HealthCheckService,
  TypeOrmHealthIndicator,
  MemoryHealthIndicator,
  DiskHealthIndicator,
} from '@nestjs/terminus';
import { HealthCheckResult } from '@nestjs/terminus';

@Controller('health')
export class HealthController {
  constructor(
    private readonly health: HealthCheckService,
    private readonly db: TypeOrmHealthIndicator,
    private readonly memory: MemoryHealthIndicator,
    private readonly disk: DiskHealthIndicator,
  ) {}

  // 基础健康检查
  @Get()
  @HealthCheck()
  async check(): Promise<HealthCheckResult> {
    return this.health.check([
      // 检查数据库连接
      () => this.db.pingCheck('database'),
    ]);
  }

  // 详细健康检查
  @Get('detailed')
  @HealthCheck()
  async detailedCheck(): Promise<HealthCheckResult> {
    return this.health.check([
      // 数据库
      () => this.db.pingCheck('database', { timeout: 3000 }),

      // 内存使用（不超过 500MB）
      () => this.memory.checkRSS('memory_rss', 500 * 1024 * 1024),

      // 磁盘使用（不超过 80%）
      () => this.disk.checkStorage('disk', {
        thresholdPercent: 0.8,
        path: '/',
      }),

      // 自定义健康检查
      async () => ({
        redis: {
          status: 'up',
          latency: await this.checkRedisLatency(),
        },
      }),
    ]);
  }

  // 就绪检查（Kubernetes readinessProbe）
  @Get('ready')
  @HealthCheck()
  async readiness(): Promise<HealthCheckResult> {
    return this.health.check([
      () => this.db.pingCheck('database'),
      // 检查其他依赖服务
      async () => ({
        external_api: {
          status: 'up',
        },
      }),
    ]);
  }

  // 存活检查（Kubernetes livenessProbe）
  @Get('live')
  @HealthCheck()
  async liveness(): Promise<HealthCheckResult> {
    return this.health.check([]);
  }

  private async checkRedisLatency(): Promise<number> {
    const start = Date.now();
    // 实际检查 Redis 延迟
    return Date.now() - start;
  }
}
```
