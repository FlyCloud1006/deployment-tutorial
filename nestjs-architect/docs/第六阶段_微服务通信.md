## 1️⃣ 微服务架构概述

### 1.1 什么是微服务

```
┌─────────────────────────────────────────────────────────────┐
│                    微服务架构 vs 单体架构                      │
├────────────────────────────┬────────────────────────────────┤
│       单体架构              │          微服务架构              │
├────────────────────────────┼────────────────────────────────┤
│  所有模块在一个应用中        │  每个服务独立部署                │
│  部署简单                   │  部署复杂（需要服务治理）         │
│  内部调用无网络开销          │  网络调用，引入延迟               │
│  技术栈统一                 │  技术栈可以不同                   │
│  扩展困难（整体扩展）        │  按需扩展（哪些服务压力大扩哪些）   │
│  故障隔离性差              │  故障隔离性好                     │
│  开发团队协同困难（代码冲突）  │  团队独立开发（接口约定）         │
└────────────────────────────┴────────────────────────────────┘
```

### 1.2 何时使用微服务

```
✅ 适合微服务的场景：
  - 大型团队（> 10 人并行开发）
  - 需要独立扩展不同模块
  - 需要使用不同技术栈
  - 需要高可用、高并发
  - 业务模块边界清晰

❌ 不适合微服务的场景：
  - 小型应用（< 5 人团队）
  - 业务边界不清晰
  - 团队缺乏 DevOps 能力
  - 需要强一致性（分布式事务难题）
```

## 2️⃣ NestJS 微服务模式

### 2.1 创建微服务应用

```typescript
// examples/stage6-microservices/01-nestjs-microservice/main.ts

import { NestFactory } from '@nestjs/core';
import { MicroserviceOptions, Transport } from '@nestjs/microservices';
import { UserServiceModule } from './user-service.module';

async function bootstrap() {
  // 方式1：创建 TCP 微服务
  const app = await NestFactory.createMicroservice<MicroserviceOptions>(
    UserServiceModule,
    {
      transport: Transport.TCP,
      options: {
        host: '0.0.0.0',
        port: 3001,
      },
    },
  );

  await app.listen();
  console.log('用户服务已启动: TCP 0.0.0.0:3001');
}

bootstrap();
```

### 2.2 创建 Hybrid Application（同时支持 HTTP + 微服务）

```typescript
// 同时支持 HTTP 和 gRPC
import { NestFactory } from '@nestjs/core';
import { MicroserviceOptions, Transport } from '@nestjs/microservices';
import { AppModule } from './app.module';

async function bootstrap() {
  const app = await NestFactory.create(AppModule);

  // 连接 gRPC 微服务
  app.connectMicroservice<MicroserviceOptions>({
    transport: Transport.GRPC,
    options: {
      package: 'user',
      protoPath: './proto/user.proto',
      url: 'localhost:5001',
    },
  });

  // 连接 Redis 消息队列
  app.connectMicroservice({
    transport: Transport.REDIS,
    options: {
      host: 'localhost',
      port: 6379,
    },
  });

  await app.startAllMicroservices();
  await app.listen(3000);
  console.log('混合应用已启动: HTTP:3000, gRPC:5001, Redis');
}

bootstrap();
```

## 3️⃣ gRPC 通信

### 3.1 Protocol Buffers 定义

```protobuf
// examples/stage6-microservices/02-grpc/proto/user.proto

syntax = "proto3";

package user;

// 用户服务
service UserService {
  // 获取用户
  rpc GetUser(GetUserRequest) returns (GetUserResponse);
  // 创建用户
  rpc CreateUser(CreateUserRequest) returns (CreateUserResponse);
  // 批量获取用户
  rpc BatchGetUsers(BatchGetUsersRequest) returns (BatchGetUsersResponse);
  // 流式获取用户（服务端推送）
  rpc StreamUsers(GetUserRequest) returns (stream GetUserResponse);
  // 流式创建（客户端流）
  rpc BatchCreateUsers(stream CreateUserRequest) returns (BatchCreateUsersResponse);
}

// 消息定义
message User {
  int32 id = 1;
  string username = 2;
  string email = 3;
  int32 age = 4;
  string created_at = 5;
}

message GetUserRequest {
  int32 id = 1;
}

message GetUserResponse {
  User user = 1;
}

message CreateUserRequest {
  string username = 1;
  string email = 2;
  int32 age = 3;
}

message CreateUserResponse {
  User user = 1;
}

message BatchGetUsersRequest {
  repeated int32 ids = 1;
}

message BatchGetUsersResponse {
  repeated User users = 1;
}

message BatchCreateUsersResponse {
  int32 created_count = 1;
  repeated User users = 2;
}
```

### 3.2 gRPC 控制器

```typescript
// examples/stage6-microservices/02-grpc/user.grpc.controller.ts

import { Controller } from '@nestjs/common';
import { GrpcMethod, GrpcStreamMethod } from '@nestjs/microservices';
import { Observable, Subject } from 'rxjs';
import { readStream } from 'rxjs/internal/operators';

@Controller()
export class UserGrpcController {
  private users = [
    { id: 1, username: '张三', email: 'zhangsan@example.com', age: 25, createdAt: '2026-01-01' },
    { id: 2, username: '李四', email: 'lisi@example.com', age: 30, createdAt: '2026-01-02' },
  ];

  // 普通 RPC
  @GrpcMethod('UserService', 'GetUser')
  getUser({ id }: { id: number }) {
    const user = this.users.find(u => u.id === id);
    return { user: user || null };
  }

  @GrpcMethod('UserService', 'CreateUser')
  createUser({ username, email, age }: { username: string; email: string; age: number }) {
    const newUser = {
      id: this.users.length + 1,
      username,
      email,
      age,
      createdAt: new Date().toISOString(),
    };
    this.users.push(newUser);
    return { user: newUser };
  }

  // 批量获取
  @GrpcMethod('UserService', 'BatchGetUsers')
  batchGetUsers({ ids }: { ids: number[] }) {
    const users = this.users.filter(u => ids.includes(u.id));
    return { users };
  }

  // 服务端流式响应
  @GrpcMethod('UserService', 'StreamUsers')
  streamUsers({ id }: { id: number }): Observable<any> {
    const subject = new Subject<any>();

    const user = this.users.find(u => u.id === id);
    if (user) {
      subject.next({ user });
      setTimeout(() => subject.next({ user: { ...user, age: user.age + 1 } }), 1000);
      setTimeout(() => subject.next({ user: { ...user, age: user.age + 2 } }), 2000);
      setTimeout(() => subject.complete(), 3000);
    } else {
      subject.complete();
    }

    return subject.asObservable();
  }

  // 客户端流式接收
  @GrpcStreamMethod('UserService', 'BatchCreateUsers')
  batchCreateUsers(dataStream: Observable<any>): Observable<any> {
    const createdUsers: any[] = [];

    return new Observable(observer => {
      dataStream.subscribe({
        next: (data) => {
          const newUser = {
            id: this.users.length + createdUsers.length + 1,
            ...data,
            createdAt: new Date().toISOString(),
          };
          createdUsers.push(newUser);
        },
        error: (err) => observer.error(err),
        complete: () => {
          this.users.push(...createdUsers);
          observer.next({ createdCount: createdUsers.length, users: createdUsers });
          observer.complete();
        },
      });
    });
  }
}
```

### 3.3 gRPC 客户端调用

```typescript
// examples/stage6-microservices/02-grpc/grpc-client.module.ts

import { Module } from '@nestjs/common';
import { ClientsModule, Transport } from '@nestjs/microservices';
import { UserServiceClient } from './user.service.client';
import { OtherService } from './other.service';

@Module({
  imports: [
    ClientsModule.register([
      {
        name: 'USER_SERVICE',           // 注入 Token
        transport: Transport.GRPC,
        options: {
          package: 'user',
          protoPath: './proto/user.proto',
          url: 'localhost:5001',
        },
      },
    ]),
  ],
  providers: [UserServiceClient, OtherService],
  exports: [UserServiceClient],
})
export class GrpcClientModule {}

// 客户端服务
import { Inject, Injectable } from '@nestjs/common';
import { ClientProxy } from '@nestjs/microservices';
import { Observable } from 'rxjs';

@Injectable()
export class UserServiceClient {
  constructor(@Inject('USER_SERVICE') private readonly client: ClientProxy) {}

  // 调用 gRPC 方法
  getUser(id: number): Observable<{ user: any }> {
    return this.client.send({ service: 'user', cmd: 'GetUser' }, { id });
  }

  createUser(data: { username: string; email: string; age: number }): Observable<any> {
    return this.client.send({ service: 'user', cmd: 'CreateUser' }, data);
  }

  // 当微服务方法名与客户端调用名不同时
  getUserCustom(data: any): Observable<any> {
    return this.client.send({ service: 'user', method: 'getUser' }, data);
  }
}

// 使用
@Injectable()
export class OtherService {
  constructor(private userClient: UserServiceClient) {}

  async getUserWithFallback(id: number) {
    try {
      return await firstValueFrom(this.userClient.getUser(id));
    } catch (e) {
      // gRPC 调用失败，降级处理
      return { user: { id, username: '降级用户' } };
    }
  }
}
```

## 4️⃣ RabbitMQ 消息队列

### 4.1 RabbitMQ 概念

```
┌─────────────────────────────────────────────────────────────┐
│                    RabbitMQ 架构                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Producer ──┐                                              │
│             ├──→ Exchange ──→ Binding ──→ Queue ──→ Consumer│
│  Producer ──┘        │                                    │
│                      │ (Routing Key 匹配)                   │
│  Consumer ◄─── Queue ◄──────────────────────────────────┘  │
│                                                             │
│  ┌─────────────────────────────────────────────────────────┐│
│  │ 交换机类型：                                             ││
│  │ - direct: 精确匹配 routing key                          ││
│  │ - topic:  通配符匹配（*.user.#）                        ││
│  │ - fanout: 广播给所有队列                                 ││
│  │ - headers: 基于消息头匹配                                 ││
│  └─────────────────────────────────────────────────────────┘│
│                                                             │
│  消息确认机制：                                             │
│  - ACK：手动确认，消费成功后确认                             │
│  - NACK：拒绝消息，可重新入队或丢弃                         │
│  - Auto ACK：自动确认（收到即确认，不推荐）                   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 4.3 生产者（发送消息）

```typescript
// examples/stage6-microservices/03-rabbitmq/producer.service.ts

import { Injectable, Inject } from '@nestjs/common';
import { ClientProxy } from '@nestjs/microservices';
import { Observable } from 'rxjs';

@Injectable()
export class ProducerService {
  constructor(@Inject('RABBITMQ_SERVICE') private client: ClientProxy) {}

  // ========== 发送消息到队列 ==========

  // 方式1：发送后不等待响应（Fire-and-Forget）
  emitOrderCreated(order: { id: number; userId: number; total: number }) {
    this.client.emit('order.created', order);  // 自动序列化
  }

  // 方式2：发送后等待响应（Request-Reply）
  sendEmailNotification(data: { userId: number; template: string }): Observable<{ success: boolean }> {
    return this.client.send({ cmd: 'email.send' }, data);
  }

  // ========== 消息模式 ==========

  // 路由模式（Routing Pattern）
  sendToRoutingKey(routingKey: string, data: any) {
    // 需要配合 @MessagePattern({ cmd: 'routingPattern' }) 使用
    this.client.emit({ routingKey }, data);
  }

  // 带 headers 的消息
  sendWithHeaders(headers: Record<string, string>, data: any) {
    this.client.emit('event.with.headers', data); // headers 通过 metadata 传递
  }
}
```

### 4.4 消费者（接收消息）

```typescript
// examples/stage6-microservices/03-rabbitmq/consumer.service.ts

import { Controller } from '@nestjs/common';
import { MessagePattern, Payload, Ctx, RmqContext, EventPattern } from '@nestjs/microservices';
import { RmqContext as RabbitMQContext } from '@nestjs/microservices';

@Controller()
export class ConsumerController {
  // ========== 事件驱动（@EventPattern）==========
  // 用于 fire-and-forget 场景

  @EventPattern('order.created')
  handleOrderCreated(@Payload() data: { id: number; userId: number; total: number }) {
    console.log('收到订单创建事件:', data);
    // 执行业务逻辑（如：发送通知、更新统计等）
  }

  // ========== 请求-响应模式（@MessagePattern）==========
  // 用于需要返回结果的场景

  @MessagePattern({ cmd: 'email.send' })
  handleSendEmail(
    @Payload() data: { userId: number; template: string },
    @Ctx() context: RabbitMQContext,
  ) {
    const originalMsg = context.getMessage();
    console.log('收到邮件发送请求:', data);

    // 业务逻辑
    const success = this.sendEmail(data);

    // 手动 ACK（因为 noAck: false）
    const channel = context.getChannelRef();
    channel.ack(originalMsg);

    return { success };
  }

  // ========== 带错误处理的消费 ==========

  @MessagePattern({ cmd: 'order.process' })
  async handleOrderProcess(
    @Payload() data: { orderId: number },
    @Ctx() context: RabbitMQContext,
  ) {
    try {
      const result = await this.processOrder(data.orderId);

      // 成功：ACK
      const channel = context.getChannelRef();
      channel.ack(context.getMessage());

      return { success: true, result };
    } catch (error) {
      // 失败：NACK，消息重新入队
      const channel = context.getChannelRef();
      channel.nack(context.getMessage(), false, true);  // 第三个参数：requeue
    }
  }

  // ========== 死信队列处理 ==========

  @MessagePattern({ cmd: 'dead.letter' })
  handleDeadLetter(@Payload() data: any, @Ctx() context: RabbitMQContext) {
    console.log('收到死信消息:', data);
    // 记录日志、上报监控系统
    // 人工干预或自动修复
  }

  private sendEmail(data: { userId: number; template: string }): boolean {
    console.log(`发送邮件给用户 ${data.userId}, 模板: ${data.template}`);
    return true;
  }

  private async processOrder(orderId: number): Promise<string> {
    console.log(`处理订单 ${orderId}`);
    return `Order ${orderId} processed`;
  }
}
```

## 5️⃣ Kafka 消息流

### 5.1 Kafka 概念

```
┌─────────────────────────────────────────────────────────────┐
│                    Kafka 架构                                 │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Producer ──→ Topic: order-events ──→ Consumer Group A      │
│                     │ (3 个 Partition)      ├──→ Consumer 1   │
│                     ├──→ Partition 0         ├──→ Consumer 2 │
│                     ├──→ Partition 1         └──→ Consumer 3 │
│                     └──→ Partition 2                         │
│                                                             │
│  Consumer Group B（独立消费，同一消息会被两个组都消费）        │
│                                                             │
│  核心概念：                                                  │
│  - Topic：消息主题                                           │
│  - Partition：分区，并行消费，同一 Partition 内有序            │
│  - Consumer Group：消费者组，组内负载均衡，组间广播             │
│  - Offset：消费位移，记录消费到哪里了                         │
│  - Retention：消息保留时间（默认7天）                         │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 5.3 Kafka 生产者

```typescript
// examples/stage6-microservices/04-kafka/kafka.producer.ts

import { Injectable, Inject } from '@nestjs/common';
import { ClientKafka } from '@nestjs/microservices';

@Injectable()
export class KafkaProducerService {
  constructor(@Inject('KAFKA_SERVICE') private client: ClientKafka) {}

  // ========== 发送消息 ==========

  // 单条发送
  async sendOrderCreated(order: { id: number; userId: number; total: number }) {
    // key 用于分区：相同 key 的消息发送到同一 Partition
    return this.client.emit('order.created', {
      key: order.id.toString(),
      value: order,
      headers: {
        'correlation-id': '12345',
        'source': 'order-service',
      },
    });
  }

  // 批量发送
  async sendBatch(topic: string, messages: any[]) {
    const record = messages.map(msg => ({
      key: msg.id?.toString(),
      value: msg,
    }));

    return this.client.emit(topic, record);
  }

  // 发送并等待 ACK（幂等生产者）
  async sendWithAck(topic: string, message: any): Promise<boolean> {
    return new Promise((resolve) => {
      this.client.emit(topic, message).subscribe({
        next: (result) => resolve(true),
        error: (err) => {
          console.error('Kafka 发送失败:', err);
          resolve(false);
        },
      });
    });
  }
}
```

### 5.4 Kafka 消费者

```typescript
// examples/stage6-microservices/04-kafka/kafka.consumer.ts

import { Controller, OnModuleInit, OnModuleDestroy } from '@nestjs/common';
import { MessagePattern, Payload, Context, KafkaContext } from '@nestjs/microservices';
import { KafkaMessage } from 'kafkajs';

@Controller()
export class KafkaConsumerController implements OnModuleInit, OnModuleDestroy {
  private messageHandlers: Map<string, Function> = new Map();

  // ========== 消息处理 ==========

  @MessagePattern('order.created')
  handleOrderCreated(
    @Payload() message: any,
    @Context() context: KafkaContext,
  ) {
    const originalMessage = context.getMessage();
    const { key, value, partition, offset, timestamp } = originalMessage;

    console.log(`收到订单消息: key=${key}, partition=${partition}, offset=${offset}`);
    console.log('消息内容:', value);

    try {
      // 业务逻辑
      this.processOrder(value);

      // 手动提交 offset（可选，Kafka 默认 auto-commit）
      // context.getConsumer().commitOffsets([{ topic: 'order.created', partition, offset }]);
    } catch (error) {
      console.error('处理消息失败:', error);
      // 可选：发送到死信 Topic
    }
  }

  @MessagePattern('user.registered')
  handleUserRegistered(
    @Payload() message: any,
    @Context() context: KafkaContext,
  ) {
    console.log('收到用户注册事件:', message);
    // 发送欢迎邮件、初始化用户数据等
  }

  // ========== 分区处理 ==========

  @MessagePattern('click.events')
  handleClickEvents(
    @Payload() messages: KafkaMessage[],
    @Context() context: KafkaContext,
  ) {
    // 批量处理（Kafka 支持批量消费）
    console.log(`收到 ${messages.length} 条点击事件`);

    messages.forEach(msg => {
      console.log('处理:', msg.value);
    });
  }

  // ========== 消费者组示例 ==========

  @MessagePattern('notification.push')
  handleNotification(
    @Payload() message: any,
    @Context() context: KafkaContext,
  ) {
    // 同一 Consumer Group 内的多个实例会负载均衡
    // 不同 Consumer Group 的实例都会收到消息
    console.log(`Consumer 实例处理通知: ${message.userId}`);
  }

  private processOrder(order: { id: number; userId: number; total: number }) {
    console.log(`处理订单 ${order.id}, 金额: ${order.total}`);
  }

  onModuleInit() {
    console.log('Kafka 消费者已初始化');
  }

  onModuleDestroy() {
    console.log('Kafka 消费者已关闭');
  }
}
```

### 6.1 WebSocket 网关

```typescript
// examples/stage6-microservices/05-websocket/chat.gateway.ts

import {
  WebSocketGateway,
  WebSocketServer,
  SubscribeMessage,
  OnGatewayInit,
  OnGatewayConnection,
  OnGatewayDisconnect,
  ConnectedSocket,
  MessageBody,
} from '@nestjs/websockets';
import { Server, Socket } from 'socket.io';
import { Logger } from '@nestjs/common';

@WebSocketGateway({
  cors: {
    origin: ['http://localhost:3000'],
    credentials: true,
  },
  namespace: '/chat',          // 命名空间
})
export class ChatGateway implements OnGatewayInit, OnGatewayConnection, OnGatewayDisconnect {
  @WebSocketServer()
  server: Server;

  private readonly logger = new Logger(ChatGateway.name);

  afterInit(server: Server) {
    this.logger.log('WebSocket 网关已初始化');
  }

  handleConnection(client: Socket) {
    this.logger.log(`客户端连接: ${client.id}`);
  }

  handleDisconnect(client: Socket) {
    this.logger.log(`客户端断开: ${client.id}`);
  }

  // ========== 客户端 → 服务端 ==========

  @SubscribeMessage('chat:join')
  handleJoinRoom(
    @ConnectedSocket() client: Socket,
    @MessageBody() data: { roomId: string; username: string },
  ) {
    const { roomId, username } = data;

    // 加入房间
    client.join(`room:${roomId}`);

    // 通知房间内其他人
    client.to(`room:${roomId}`).emit('chat:system', {
      message: `${username} 加入了房间`,
      roomId,
    });

    // 返回确认
    return { event: 'chat:join', data: { success: true, roomId } };
  }

  @SubscribeMessage('chat:leave')
  handleLeaveRoom(
    @ConnectedSocket() client: Socket,
    @MessageBody() data: { roomId: string; username: string },
  ) {
    const { roomId, username } = data;

    client.leave(`room:${roomId}`);
    client.to(`room:${roomId}`).emit('chat:system', {
      message: `${username} 离开了房间`,
      roomId,
    });

    return { event: 'chat:leave', data: { success: true } };
  }

  @SubscribeMessage('chat:message')
  handleChatMessage(
    @ConnectedSocket() client: Socket,
    @MessageBody() data: { roomId: string; username: string; content: string },
  ) {
    const { roomId, username, content } = data;
    const timestamp = new Date().toISOString();

    // 广播给房间内所有人（包括发送者）
    this.server.to(`room:${roomId}`).emit('chat:message', {
      id: `${Date.now()}-${Math.random().toString(36).slice(2)}`,
      username,
      content,
      timestamp,
      roomId,
    });

    return { event: 'chat:message', data: { sent: true } };
  }

  @SubscribeMessage('chat:typing')
  handleTyping(
    @ConnectedSocket() client: Socket,
    @MessageBody() data: { roomId: string; username: string },
  ) {
    const { roomId, username } = data;

    // 广播给房间内其他人（除了发送者）
    client.to(`room:${roomId}`).emit('chat:typing', { username });
  }
}
```

## 7️⃣ 消息队列设计模式

### 7.1 死信队列（Dead Letter Queue）

```
┌─────────────────────────────────────────────────────────────┐
│                    死信队列处理流程                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  正常队列 ──→ Consumer 处理失败 ──→ 重试 N 次 ──→            │
│              │ (NACK + requeue: false)                      │
│              ↓                                              │
│           死信交换机（DLX） ──→ 死信队列 ──→ 人工处理        │
│                                                             │
│  配置：                                                     │
│  x-dead-letter-exchange: dlx.exchange                        │
│  x-dead-letter-routing-key: dlx.order.process               │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 7.2 延迟队列（Delay Queue）

```typescript
// 实现订单 30 分钟未支付自动取消
@EventPattern('order.created')
async handleOrderCreated(data: { orderId: number; userId: number }) {
  const delayMs = 30 * 60 * 1000; // 30 分钟

  // 方式1：使用 RabbitMQ 延迟插件
  await this.client.emit('order.delay.cancel', {
    ...data,
    _delay: delayMs,  // 消息将延迟 30 分钟后投递
  });

  // 方式2：使用 Redis ZSet 实现延迟队列
  const executeAt = Date.now() + delayMs;
  await this.redis.zadd('delayed:orders', executeAt, JSON.stringify(data));
}
```

### 7.3 消息幂等性

```typescript
// 消费者端：保证消息处理幂等性
@MessagePattern({ cmd: 'order.process' })
async handleOrderProcess(@Payload() data: { orderId: number }, @Ctx() context: RabbitMQContext) {
  const { orderId } = data;
  const processedKey = `processed:order:${orderId}`;

  // 1. 检查是否已处理
  const alreadyProcessed = await this.redis.get(processedKey);
  if (alreadyProcessed) {
    console.log(`订单 ${orderId} 已处理过，跳过`);
    const channel = context.getChannelRef();
    channel.ack(context.getMessage()); // ACK，消息不再重试
    return { success: true, idempotent: true };
  }

  // 2. 处理订单
  try {
    await this.processOrder(orderId);

    // 3. 标记为已处理（设置过期时间）
    await this.redis.set(processedKey, '1', 'EX', 86400); // 24 小时过期

    const channel = context.getChannelRef();
    channel.ack(context.getMessage());
  } catch (error) {
    const channel = context.getChannelRef();
    channel.nack(context.getMessage(), false, true); // 失败重试
  }
}
```
