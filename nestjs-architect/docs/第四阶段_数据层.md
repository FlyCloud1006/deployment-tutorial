## 0️⃣ 先修导读：数据层的三个核心问题

> 数据层 = 数据库 + 缓存。本阶段解决三个核心问题：**数据怎么存**（ORM）、**缓存怎么用**（Redis）、**缓存出问题了怎么办**（穿透/击穿/雪崩）。

---

### 🔍 1. 数据怎么存？ORM 是什么？

#### 🍜 生活类比：图书馆 vs 书架

**没有 ORM 时**（直接写 SQL）：
- 就像去图书馆，你要自己记住书在哪个柜子、哪一层、第几排
- SQL 语句就是"去3号柜第2层找《Node.js实战》"

**有 ORM 时**：
- 就像告诉图书管理员"我要借《Node.js实战》"
- 管理员帮你找到，不用你关心具体放在哪里

**ORM = Object Relational Mapping（对象关系映射）**：把数据库的表（Table）映射成编程语言里的对象（Object）。
- 你操作对象，ORM 自动生成 SQL
- 不需要手写 `SELECT * FROM users WHERE id = 1`
- 写 `userRepo.findById(1)` 就行

#### 常见 ORM 对比

| ORM | 特点 | 适用场景 |
|-----|------|---------|
| TypeORM | NestJS 官方推荐，支持 MySQL/PostgreSQL/SQLite | **NestJS首选** |
| Prisma | 新型 ORM，Schema 驱动，类型安全 | TypeScript 项目 |
| Sequelize | 老牌 ORM，Express 时代主流 | 兼容性要求高的项目 |

### 🔍 2. 为什么要用 Redis 缓存？

#### ⚡ 速度对比

| 操作 | MySQL | Redis |
|------|-------|-------|
| 读取一个用户 | ~10-50ms | ~0.1-1ms |
| 并发能力 | 中（受连接池限制）| 极高（单实例 10万+ QPS）|

**Redis 为什么这么快？**
- 数据存在**内存**里（不是硬盘）
- 用 C 语言写的，执行效率高
- 单线程（无锁竞争），事件循环驱动

**但 Redis 不能替代数据库**：数据放在内存里，断电会丢（除非开启持久化）。所以 Redis 是**缓存层**，MySQL 才是**持久化层**。

#### 什么时候该用 Redis？

- 数据**读多写少**（如商品详情、用户信息）
- 需要**高性能**（毫秒级响应）
- 需要**持久化会话**（Session 存储）
- 需要**分布式锁**（跨服务同步）
- 需要**计数器/排行榜**（ZSet）

### 🔍 3. 缓存三大的问题（穿透/击穿/雪崩）

> **重要！这是面试必问内容，也是生产环境最容易出问题的点。**

#### 🛒 双十一场景理解

**缓存穿透**：恶意用户用不存在的商品 ID 疯狂请求，每次都绕过缓存直接查数据库。
- 危害：数据库被打死
- 解决：布隆过滤器 + 缓存空值

**缓存击穿**：热门商品（比如 iphone 打折）的缓存刚好过期，大量请求同时涌入数据库。
- 危害：数据库被打死
- 解决：分布式锁 + 逻辑过期 + 永不过期

**缓存雪崩**：双十一零点Thousands商品同时过期，所有请求同时打向数据库。
- 危害：数据库被打死
- 解决：随机过期时间 + 多级缓存 + 接口限流

**一句话总结**：三个问题的本质都是**缓存失效 → 数据库压力大**。区别在于触发条件不同。

```typescript
// examples/stage4-orm-redis/01-typeorm/user.repository.ts

import { Injectable } from '@nestjs/common';
import { InjectRepository } from '@nestjs/typeorm';
import { Repository, Like, In, Between } from 'typeorm';
import { User } from './user.entity';

@Injectable()
export class UserRepository {
  constructor(@InjectRepository(User) private readonly repo: Repository<User>) {}

  async create(dto: Partial<User>): Promise<User> {
    return this.repo.save(this.repo.create(dto));
  }

  async findById(id: number): Promise<User | null> {
    return this.repo.findOne({ where: { id } });
  }

  async findByEmail(email: string): Promise<User | null> {
    return this.repo.findOne({ where: { email } });
  }

  async findAll(params: {
    page?: number; limit?: number; status?: string; keyword?: string
  }): Promise<{ list: User[]; total: number }> {
    const { page = 1, limit = 10, status, keyword } = params;
    const where: any = {};
    if (status) where.status = status;
    if (keyword) where.username = Like(`%${keyword}%`);

    const [list, total] = await this.repo.findAndCount({
      where,
      order: { createdAt: 'DESC' },
      skip: (page - 1) * limit,
      take: limit,
    });
    return { list, total };
  }

  async update(id: number, dto: Partial<User>): Promise<User> {
    await this.repo.update(id, dto);
    return this.findById(id);
  }

  async softDelete(id: number): Promise<void> {
    await this.repo.softDelete(id);
  }

  async countByStatus(): Promise<{ status: string; count: number }[]> {
    return this.repo
      .createQueryBuilder('user')
      .select('user.status', 'status')
      .addSelect('COUNT(*)', 'count')
      .groupBy('user.status')
      .getRawMany();
  }
}
```

### 1.3 QueryBuilder 高级查询

```typescript
// examples/stage4-orm-redis/01-typeorm/query-builder.ts

import { Injectable } from '@nestjs/common';
import { InjectRepository } from '@nestjs/typeorm';
import { Repository } from 'typeorm';

@Injectable()
export class OrderRepository {
  constructor(@InjectRepository(Order) private readonly repo: Repository<Order>) {}

  async findOrdersWithUser(params: {
    status?: string; minAmount?: number;
    startDate?: Date; endDate?: Date;
    page?: number; limit?: number;
  }) {
    const { page = 1, limit = 10, status, minAmount, startDate, endDate } = params;
    const query = this.repo
      .createQueryBuilder('order')
      .leftJoinAndSelect('order.user', 'user')
      .leftJoinAndSelect('order.items', 'item')
      .where('order.deletedAt IS NULL');

    if (status) query.andWhere('order.status = :status', { status });
    if (minAmount !== undefined) query.andWhere('order.totalAmount >= :minAmount', { minAmount });
    if (startDate) query.andWhere('order.createdAt >= :startDate', { startDate });
    if (endDate) query.andWhere('order.createdAt <= :endDate', { endDate });

    const [list, total] = await query
      .orderBy('order.createdAt', 'DESC')
      .skip((page - 1) * limit)
      .take(limit)
      .getManyAndCount();

    return { list, total, page, limit, totalPages: Math.ceil(total / limit) };
  }

  async findUsersWithRecentOrders(days: number = 30) {
    const subQuery = this.repo
      .createQueryBuilder('order')
      .select('order.userId')
      .where(`order.createdAt > DATE_SUB(NOW(), INTERVAL :days DAY)`, { days })
      .groupBy('order.userId')
      .having('COUNT(*) > 5');

    return this.repo
      .createQueryBuilder('user')
      .where(`user.id IN (${subQuery.getQuery()})`)
      .setParameters(subQuery.getParameters())
      .getMany();
  }
}
```

```typescript
import { Injectable, OnModuleInit, OnModuleDestroy } from '@nestjs/common';
import { PrismaClient } from '@prisma/client';

@Injectable()
export class PrismaService extends PrismaClient implements OnModuleInit, OnModuleDestroy {
  async onModuleInit() { await this.$connect(); }
  async onModuleDestroy() { await this.$disconnect(); }

  async transaction<T>(fn: (prisma: PrismaClient) => Promise<T>): Promise<T> {
    return this.$transaction(fn);
  }
}

@Injectable()
export class UserRepository {
  constructor(private readonly prisma: PrismaService) {}

  async findByEmail(email: string) {
    return this.prisma.user.findUnique({ where: { email } });
  }

  async findOrdersWithUsers(params: { page?: number; limit?: number; status?: string }) {
    const { page = 1, limit = 10, status } = params;
    const where = status ? { status: status as any } : {};

    const [list, total] = await this.prisma.$transaction([
      this.prisma.order.findMany({
        where,
        include: { user: true, items: true },
        orderBy: { createdAt: 'desc' },
        skip: (page - 1) * limit,
        take: limit,
      }),
      this.prisma.order.count({ where }),
    ]);
    return { list, total, page, limit, totalPages: Math.ceil(total / limit) };
  }

  async getUserStatistics(userId: number) {
    const [orderCount, commentCount, totalSpent] = await this.prisma.$transaction([
      this.prisma.order.count({ where: { userId } }),
      this.prisma.comment.count({ where: { userId } }),
      this.prisma.order.aggregate({
        where: { userId, status: 'completed' },
        _sum: { totalAmount: true },
      }),
    ]);
    return { orderCount, commentCount, totalSpent: totalSpent._sum.totalAmount || 0 };
  }
}
```

### 3.2 String 类型

```typescript
// examples/stage4-orm-redis/03-redis-data-types/string-ops.ts

import { Injectable } from '@nestjs/common';
import { RedisService } from '@nestjs/cache-manager';

@Injectable()
export class RedisStringService {
  constructor(private redis: RedisService) {}

  async set(key: string, value: string, ttlSeconds?: number) {
    if (ttlSeconds) {
      await this.redis.set(key, value, ttlSeconds);
    } else {
      await this.redis.set(key, value);
    }
  }

  async get(key: string): Promise<string | null> {
    return this.redis.get(key);
  }

  async del(key: string): Promise<void> {
    await this.redis.del(key);
  }

  async incr(key: string): Promise<number> { return this.redis.incr(key); }
  async decr(key: string): Promise<number> { return this.redis.decr(key); }
  async incrBy(key: string, amount: number): Promise<number> { return this.redis.incrby(key, amount); }

  // 限流
  async checkRateLimit(userId: string, maxRequests: number, windowSeconds: number): Promise<boolean> {
    const key = `rate:${userId}`;
    const current = await this.incr(key);
    if (current === 1) await this.redis.expire(key, windowSeconds);
    return current <= maxRequests;
  }

  // 分布式锁
  async acquireLock(lockKey: string, ttlSeconds: number = 30): Promise<string | null> {
    const lockValue = crypto.randomUUID();
    const acquired = await this.redis.setnx(lockKey, lockValue);
    if (acquired) { await this.redis.expire(lockKey, ttlSeconds); return lockValue; }
    return null;
  }

  async releaseLock(lockKey: string, lockValue: string): Promise<boolean> {
    const script = `if redis.call("get", KEYS[1]) == ARGV[1] then return redis.call("del", KEYS[1]) else return 0 end`;
    return (await this.redis.eval(script, [lockKey], [lockValue])) === 1;
  }

  async cacheJson<T>(key: string, data: T, ttlSeconds: number = 3600) {
    await this.set(key, JSON.stringify(data), ttlSeconds);
  }

  async getCachedJson<T>(key: string): Promise<T | null> {
    const data = await this.get(key);
    return data ? JSON.parse(data) : null;
  }
}
```

```typescript
// examples/stage4-orm-redis/03-redis-data-types/hash-ops.ts

import { Injectable } from '@nestjs/common';
import { RedisService } from '@nestjs/cache-manager';

@Injectable()
export class RedisHashService {
  constructor(private redis: RedisService) {}

  // 购物车
  async addToCart(userId: number, productId: number, quantity: number) {
    const key = `cart:${userId}`;
    await this.redis.hincrby(key, `product:${productId}`, quantity);
    await this.redis.expire(key, 7 * 24 * 3600);
  }

  async getCart(userId: number): Promise<Record<string, number>> {
    return this.redis.hgetall(`cart:${userId}`) as any;
  }

  async removeFromCart(userId: number, productId: number) {
    await this.redis.hdel(`cart:${userId}`, `product:${productId}`);
  }

  // 用户信息缓存
  async cacheUserProfile(userId: number, profile: { username: string; avatar: string; bio: string }) {
    const key = `user:profile:${userId}`;
    await this.redis.hmset(key, profile as any);
    await this.redis.expire(key, 3600);
  }

  async getUserProfile(userId: number): Promise<any | null> {
    const profile = await this.redis.hgetall(`user:profile:${userId}`);
    return Object.keys(profile).length > 0 ? profile : null;
  }
}
```

```typescript
// examples/stage4-orm-redis/03-redis-data-types/list-ops.ts

import { Injectable } from '@nestjs/common';
import { RedisService } from '@nestjs/cache-manager';

@Injectable()
export class RedisListService {
  constructor(private redis: RedisService) {}

  // Timeline（最新消息）
  async addMessage(userId: number, messageId: string) {
    const key = `timeline:${userId}`;
    await this.redis.lpush(key, messageId);
    await this.redis.ltrim(key, 0, 999);
    await this.redis.expire(key, 7 * 24 * 3600);
  }

  async getTimeline(userId: number, page: number = 1, limit: number = 20): Promise<string[]> {
    return this.redis.lrange(`timeline:${userId}`, (page - 1) * limit, (page - 1) * limit + limit - 1);
  }

  // 任务队列
  async enqueueTask(queueName: string, taskData: string) {
    await this.redis.rpush(`queue:${queueName}`, taskData);
  }

  async dequeueTask(queueName: string, timeoutSeconds: number = 0): Promise<string | null> {
    if (timeoutSeconds > 0) {
      const result = await this.redis.brpop(`queue:${queueName}`, timeoutSeconds);
      return result ? result[1] : null;
    }
    return this.redis.rpop(`queue:${queueName}`);
  }
}
```

### 3.6 ZSet 类型（排行榜）

```typescript
// examples/stage4-orm-redis/03-redis-data-types/zset-ops.ts

import { Injectable } from '@nestjs/common';
import { RedisService } from '@nestjs/cache-manager';

@Injectable()
export class RedisZSetService {
  constructor(private redis: RedisService) {}

  // 游戏排行榜
  async addScore(gameId: string, userId: string, score: number) {
    await this.redis.zadd(`leaderboard:${gameId}`, score, userId);
  }

  async getUserRank(gameId: string, userId: string): Promise<number | null> {
    const rank = await this.redis.zrevrank(`leaderboard:${gameId}`, userId);
    return rank !== null ? rank + 1 : null;
  }

  async getTopN(gameId: string, n: number = 10): Promise<{ userId: string; score: number }[]> {
    const results = await this.redis.zrevrange(`leaderboard:${gameId}`, 0, n - 1, 'WITHSCORES');
    const items: { userId: string; score: number }[] = [];
    for (let i = 0; i < results.length; i += 2) {
      items.push({ userId: results[i], score: parseFloat(results[i + 1]) });
    }
    return items;
  }

  // 销量排行
  async incrementProductSales(productId: string, quantity: number = 1) {
    await this.redis.zincrby('product:sales:daily', quantity, productId);
    await this.redis.expire('product:sales:daily', 86400 * 7);
  }

  // 延迟队列
  async scheduleOrder(orderId: string, executeAt: Date) {
    await this.redis.zadd('delayed:orders', executeAt.getTime(), orderId);
  }

  async pollExpiredOrders(): Promise<string[]> {
    return this.redis.zrangebyscore('delayed:orders', 0, Date.now());
  }
}
```

## 4️⃣ 缓存策略详解

### 4.1 缓存穿透（Cache Penetration）

```
问题：大量请求查询不存在的数据，直接打到数据库
原因：数据根本不存在，缓存永远不会命中
```

```typescript
// examples/stage4-orm-redis/04-redis-cache/cache-penetration.ts

@Injectable()
export class UserService {
  constructor(
    private readonly userRepo: UserRepository,
    private readonly redis: RedisService,
  ) {}

  // 方案1：缓存空值（简单有效）
  async findById(id: number) {
    const cacheKey = `user:${id}`;

    const cached = await this.redis.get(cacheKey);
    if (cached !== null) {
      if (cached === 'null') return null;
      return JSON.parse(cached);
    }

    const user = await this.userRepo.findById(id);

    if (user) {
      await this.redis.set(cacheKey, JSON.stringify(user), 3600);
    } else {
      // 关键：缓存空值，防止穿透（过期时间短一些）
      await this.redis.set(`null:${cacheKey}`, '1', 300);
    }

    return user;
  }
}
```

### 4.2 缓存击穿（Cache Breakdown）

```
问题：热点 key 过期瞬间，大量并发请求直接打到数据库
原因：热点数据过期，新请求同时发现缓存不存在
```

```typescript
// examples/stage4-orm-redis/04-redis-cache/cache-breakdown.ts

@Injectable()
export class UserService {
  constructor(
    private readonly userRepo: UserRepository,
    private readonly redis: RedisService,
    private readonly lock: DistributedLockService,
  ) {}

  // 方案1：分布式锁（互斥访问）
  async findByIdWithLock(id: number) {
    const cacheKey = `user:${id}`;

    const cached = await this.redis.get(cacheKey);
    if (cached) return JSON.parse(cached);

    const lockKey = `lock:${cacheKey}`;
    const lockValue = await this.lock.acquire(lockKey, 10);

    if (!lockValue) {
      // 获取锁失败，短暂等待后重试缓存
      await new Promise(resolve => setTimeout(resolve, 100));
      const retryCached = await this.redis.get(cacheKey);
      return retryCached ? JSON.parse(retryCached) : this.userRepo.findById(id);
    }

    try {
      const user = await this.userRepo.findById(id);
      if (user) await this.redis.set(cacheKey, JSON.stringify(user), 3600);
      return user;
    } finally {
      await this.lock.release(lockKey, lockValue);
    }
  }

  // 方案2：逻辑过期（推荐，适合读多写少）
  async findByIdWithLogicExpire(id: number) {
    const cacheKey = `user:${id}`;
    const cached = await this.redis.get(cacheKey);
    if (!cached) {
      const user = await this.userRepo.findById(id);
      await this.setUserCache(id, user);
      return user;
    }

    const cacheData = JSON.parse(cached);
    if (Date.now() > cacheData.logicExpireAt) {
      // 过期了：异步更新，立即返回旧数据
      this.refreshUserCacheAsync(id);
      return cacheData.user;
    }
    return cacheData.user;
  }

  async setUserCache(id: number, user: any) {
    await this.redis.set(`user:${id}`, JSON.stringify({
      user,
      logicExpireAt: Date.now() + 3600000, // 1小时逻辑过期
    }), -1);
  }

  @Cron('*/5 * * * *')
  async refreshUserCacheAsync(id: number) {
    const user = await this.userRepo.findById(id);
    if (user) await this.setUserCache(id, user);
  }
}
```

### 4.3 缓存雪崩（Cache Avalanche）

```
问题：大量缓存同时过期，导致大量请求直接打到数据库
原因：缓存集中过期、Redis 宕机
```

```typescript
// examples/stage4-orm-redis/04-redis-cache/cache-avalanche.ts

@Injectable()
export class CacheStrategyService {
  constructor(private redis: RedisService) {}

  // 方案1：随机过期时间（避免同时过期）
  async setWithJitter(key: string, value: any, baseTtlSeconds: number = 3600) {
    const jitter = Math.floor(Math.random() * baseTtlSeconds * 0.1);
    await this.redis.set(key, JSON.stringify(value), baseTtlSeconds + jitter);
  }

  // 方案2：多级缓存架构
  // L1 本地缓存（Caffeine/Guava）→ L2 Redis
  // 本地缓存扛住大部分请求，Redis 作为共享缓存

  // 方案3：Redis 持久化 + 自动故障恢复
  // 配置 RDB + AOF，保证数据不丢失

  // 方案4：接口限流 + 数据库保护
  // 缓存雪崩时，限制对数据库的直接访问
}
```

## 5️⃣ 分布式锁实现

### 5.1 Redis 分布式锁

```typescript
// examples/stage4-orm-redis/05-redis-distributed-lock/distributed-lock.ts

import { Injectable, Logger } from '@nestjs/common';
import { RedisService } from '@nestjs/cache-manager';

@Injectable()
export class DistributedLockService {
  private readonly logger = new Logger(DistributedLockService.name);

  // 推荐版：SET NX PX（原子操作）
  async acquire(key: string, ttlSeconds: number = 30): Promise<string | null> {
    const lockKey = `lock:${key}`;
    const lockValue = `${Date.now()}-${Math.random().toString(36).slice(2)}`;
    const result = await this.redis.set(lockKey, lockValue, 'PX', ttlSeconds * 1000, 'NX');
    return result ? lockValue : null;
  }

  // 释放锁（Lua 脚本保证原子性）
  async release(key: string, lockValue: string): Promise<boolean> {
    const lockKey = `lock:${key}`;
    const script = `
      if redis.call("get", KEYS[1]) == ARGV[1] then
        return redis.call("del", KEYS[1])
      else
        return 0
      end
    `;
    return (await this.redis.eval(script, [lockKey], [lockValue])) === 1;
  }

  // 续期（watch dog 机制）
  async renew(key: string, lockValue: string, ttlSeconds: number = 30): Promise<boolean> {
    const lockKey = `lock:${key}`;
    const script = `
      if redis.call("get", KEYS[1]) == ARGV[1] then
        return redis.call("pexpire", KEYS[1], ARGV[2])
      else
        return 0
      end
    `;
    return (await this.redis.eval(script, [lockKey], [lockValue, ttlSeconds * 1000])) === 1;
  }

  // 抢票系统示例
  async purchaseTicket(userId: number, ticketId: number) {
    const lockKey = `ticket:${ticketId}`;
    const lockValue = await this.acquire(lockKey, 10);

    if (!lockValue) throw new ConflictException('系统繁忙，请稍后重试');

    try {
      const ticket = await this.ticketRepo.findById(ticketId);
      if (!ticket || ticket.status !== 'available') throw new BadRequestException('票已售罄');

      ticket.status = 'sold';
      ticket.buyerId = userId;
      await this.ticketRepo.save(ticket);
      await this.orderRepo.create({ userId, ticketId });

      this.logger.log(`用户 ${userId} 成功购买票 ${ticketId}`);
      return { success: true };
    } finally {
      await this.release(lockKey, lockValue);
    }
  }
}
```

```typescript
// examples/stage4-orm-redis/06-mysql-optimization/index-design.ts

// 索引设计黄金法则：
```

## 7️⃣ Redis Key 命名规范

```
格式：{业务前缀}:{实体类型}:{实体ID}:{字段}

示例：
user:profile:1234              → 用户 1234 的 profile
order:items:5678              → 订单 5678 的商品列表
cart:9012                     → 用户 9012 的购物车
token:abcd1234                → Token abcd1234 的会话
rate:api:users:5678           → 用户 5678 的 API 限流
lock:order:1234               → 订单 1234 的分布式锁
cache:page:home              → 首页缓存
leaderboard:game:season1      → 第一赛季游戏排行榜

注意：
- Key 不要太长（占用内存，影响查询性能）
- Key 不要太短（可读性差）
- 避免使用大 Key（单 Key > 10MB 影响集群性能）
```
