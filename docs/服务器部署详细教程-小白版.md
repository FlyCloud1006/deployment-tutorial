## 一、基础知识科普

### 1.1 服务器是什么？

**服务器**就是一台永远开机的电脑，和你家里的 PC 类似，但配置更高、性能更强，放在机房里通过高速网络连接互联网。

| 对比项 | 家用电脑 | 服务器 |
|-------|---------|-------|
| 运行时间 | 按需开机 | 24小时运行 |
| 稳定性 | 一般 | 7×24小时稳定 |
| 带宽 | 家宽（上传慢） | 独享高速带宽 |
| 位置 | 家里 | 机房 |
| 系统 | Windows/Mac | Linux/Windows |

### 1.2 Linux 是什么？

Linux 是一款开源的操作系统，服务器领域90%以上都在用。它的特点：

- **免费**：不需要付费
- **稳定**：很少崩溃
- **安全**：权限管理严格
- **轻量**：占用资源少

常见 Linux 发行版：Ubuntu、CentOS、Debian

### 1.3 命令行是什么？

命令行（Terminal）是通过文字命令来操作电脑，不需要图形界面。

```
Windows: 开始菜单 → cmd → 黑色窗口
Mac: 启动台 → 终端
Linux: 右键 → 终端
```

连接服务器后的样子：
```
root@你的服务器:~#
```
- `root` 是当前用户名
- `你的服务器` 是主机名
- `#` 表示超级管理员权限

### 1.4 常用命令速查

| 命令 | 作用 | 示例 |
|-----|------|-----|
| `ls` | 查看当前目录文件 | `ls -la`（详细列表）|
| `cd` | 切换目录 | `cd /root` |
| `pwd` | 显示当前路径 | `pwd` |
| `mkdir` | 创建目录 | `mkdir project` |
| `rm` | 删除文件 | `rm -rf project` |
| `cp` | 复制文件 | `cp a.txt b.txt` |
| `mv` | 移动/重命名 | `mv a.txt b.txt` |
| `cat` | 查看文件内容 | `cat config.js` |
| `vi` / `nano` | 编辑文件 | `vi config.js` |
| `grep` | 搜索内容 | `grep "error" log.txt` |
| `curl` | 发送HTTP请求 | `curl http://baidu.com` |
| `wget` | 下载文件 | `wget https://xxx.zip` |

### 1.5 目录结构说明

Linux 目录结构：

```
/
├── bin/          # 存放可执行文件
├── etc/          # 系统配置文件
├── home/         # 用户主目录
├── root/         # 超级管理员目录
├── tmp/          # 临时文件
├── usr/          # 用户程序
├── var/          # 经常变化的数据（日志）
└── www/          # 网站文件目录（常用）
```

### 1.6 端口是什么？

**端口**就像一栋大楼的不同房间号，数据通过不同的"房间号"找到对应的服务。

| 端口号 | 常用服务 |
|-------|---------|
| 22 | SSH（远程连接）|
| 80 | HTTP（网页访问）|
| 443 | HTTPS（加密网页）|
| 3306 | MySQL数据库 |
| 3000 | Node.js服务 |
| 8080 | Java/Tomcat服务 |

## 二、服务器购买与连接

### 2.1 购买服务器

推荐服务商：
- **阿里云**：https://www.aliyun.com（国内访问快）
- **腾讯云**：https://cloud.tencent.com
- **AWS**：https://aws.amazon.com
- **Vultr**：https://www.vultr.com（国外，性价比高）

**配置推荐（新手入门）**：
- CPU: 1核
- 内存: 2GB
- 硬盘: 50GB SSD
- 系统: Ubuntu 22.04 LTS
- 带宽: 1-5Mbps

**价格参考**：约 20-60元/月

### 2.2 获取服务器 IP

购买成功后，服务商会在控制台显示：
- **公网IP**：如 `43.167.209.167`（用于访问服务器）
- **用户名**：通常是 `root`
- **密码**：或使用 SSH Key

### 2.3 连接服务器

#### Windows 用户

**方法1：使用 PowerShell**
```powershell
ssh root@43.167.209.167
```

**方法2：使用宝塔面板（可视化）**
1. 宝塔官网下载：https://www.bt.cn
2. 一键安装，自动配置环境

**方法3：使用 FinalShell（推荐）**
1. 下载 FinalShell：http://www.hostbuf.com
2. 新建连接 → 填写IP、用户名、密码
3. 界面友好，支持可视化操作

#### Mac / Linux 用户

直接打开终端：
```bash
ssh root@43.167.209.167
```

### 2.4 首次连接设置

首次登录后会提示修改 root 密码，建议设置强密码：

```
# 修改root密码
passwd root

# 创建新用户（更安全）
adduser deploy

# 给新用户sudo权限
usermod -aG sudo deploy
```

### 2.5 SSH Key 免密登录（推荐）

**步骤1：在本机生成密钥**
```bash
# Mac/Linux 在终端执行
ssh-keygen -t rsa -b 4096 -C "your_email@example.com"

# Windows PowerShell 执行相同命令
```

一路回车，会在 `~/.ssh/` 目录生成：
- `id_rsa`（私钥，要保密）
- `id_rsa.pub`（公钥，可以公开）

**步骤2：把公钥传到服务器**
```bash
ssh-copy-id root@43.167.209.167

# 或者手动复制
cat ~/.ssh/id_rsa.pub
# 复制内容，登录服务器后：
mkdir -p ~/.ssh
echo "粘贴公钥内容" >> ~/.ssh/authorized_keys
chmod 600 ~/.ssh/authorized_keys
```

**步骤3：以后登录直接**
```bash
ssh root@43.167.209.167
# 无需输入密码
```

## 三、环境搭建

### 3.1 更新系统

登录服务器后，第一件事更新系统：
```bash
# 更新软件包列表
apt update

# 升级所有软件
apt upgrade -y
```

### 3.2 安装 Node.js

Node.js 是运行 JavaScript 的环境，我们的后端项目需要它。

**方法1：通过 apt 安装（简单但版本旧）**
```bash
apt install nodejs npm -y
node -v  # 查看版本
npm -v   # 查看npm版本
```

**方法2：通过 NVM 安装（推荐，版本管理方便）**
```bash
# 安装 NVM（Node Version Manager）
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# 重新加载环境
source ~/.bashrc

# 安装最新版 Node.js
nvm install node

# 或者安装指定版本
nvm install 18
nvm install 20

# 切换版本
nvm use 20

# 验证
node -v
npm -v
```

**方法3：通过 NodeSource 安装（稳定版）**
```bash
# 安装 Node.js 20.x
curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
apt install nodejs -y
node -v
```

### 3.3 安装 PM2

PM2 是 Node.js 进程管理器，让你的服务在后台运行。

```bash
# 使用 npm 全局安装
npm install -g pm2

# 验证
pm2 -v
```

**PM2 常用命令：**
```bash
pm2 start app.js              # 启动应用
pm2 list                      # 查看所有进程
pm2 logs                      # 查看日志
pm2 restart app               # 重启应用
pm2 stop app                  # 停止应用
pm2 delete app                # 删除进程
pm2 monit                     # 监控面板
pm2 startup                   # 开机自启
pm2 save                      # 保存当前进程列表
```

### 3.4 安装 Git

Git 是版本控制工具，代码管理必备。
```bash
apt install git -y
git --version
```

**配置 Git：**
```bash
git config --global user.name "你的名字"
git config --global user.email "你的邮箱"
```

### 3.5 安装 Nginx

Nginx 是 web 服务器和反向代理服务器。

```bash
apt install nginx -y
nginx -v
```

**Nginx 常用命令：**
```bash
systemctl start nginx     # 启动
systemctl stop nginx      # 停止
systemctl restart nginx   # 重启
systemctl reload nginx    # 重新加载配置
systemctl status nginx    # 查看状态
nginx -t                  # 测试配置语法
```

### 3.6 安装 MySQL

MySQL 是最流行的开源关系型数据库。

```bash
apt install mysql-server -y

# 启动并设置开机启动
systemctl start mysql
systemctl enable mysql

# 安全初始化
mysql_secure_installation

# 登录 MySQL
mysql -u root -p
```

**MySQL 常用命令：**
```sql
-- 查看数据库
SHOW DATABASES;

-- 创建数据库
CREATE DATABASE rbac_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 创建用户并授权
```

Redis 是内存数据库，用于缓存和会话存储。


## 四、Nginx 详解

### 4.1 Nginx 是什么？

Nginx 是一款高性能的 web 服务器软件，主要功能：

1. **静态文件服务**：直接返回 HTML、CSS、JS、图片
2. **反向代理**：把请求转发到后端服务
3. **负载均衡**：分发请求到多台服务器
4. **SSL Termination**：处理 HTTPS 加密解密

### 4.2 工作原理

```
用户请求 → Nginx → 判断规则
              ↓
         静态文件？→ 直接返回
              ↓
         反向代理？→ 转发到后端服务
```

### 4.3 配置文件结构

Nginx 主配置文件：`/etc/nginx/nginx.conf`

网站配置目录：`/etc/nginx/sites-available/`（可用站点）
启用的站点：`/etc/nginx/sites-enabled/`（软链接）

**创建新站点配置：**
```bash
# 编辑配置
vi /etc/nginx/sites-available/myapp

# 创建软链接启用
ln -s /etc/nginx/sites-available/myapp /etc/nginx/sites-enabled/

# 测试配置
nginx -t

# 重载配置
systemctl reload nginx
```

### 4.4 配置示例：反向代理

假设后端 Node.js 服务运行在 `127.0.0.1:3000`：

```nginx
server {
    listen 80;
    server_name your-domain.com;  # 你的域名或IP

    # 处理静态文件（如果有）
    location / {
        root /var/www/myapp/dist;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # API 反向代理
    location /api/ {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # WebSocket 支持（如果需要）
    location /ws/ {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```

### 4.5 配置多项目路径

如果一台服务器部署多个项目：

```
http://43.167.209.167/rbac/      → Node.js:3000
http://43.167.209.167/api2/     → Node.js:3001
http://43.167.209.167/other/    → Java:8080
```

```nginx
server {
    listen 80;
    server_name 43.167.209.167;

    # RBAC 项目
    location /rbac/ {
        rewrite ^/rbac/(.*) /$1 break;
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # API 项目
    location /api2/ {
        rewrite ^/api2/(.*) /$1 break;
        proxy_pass http://127.0.0.1:3001;
        proxy_set_header Host $host;
    }
}
```

## 五、PM2 详解

### 5.1 为什么需要 PM2？

Node.js 程序直接运行 `node app.js` 时：
- 关闭终端 = 程序停止
- 崩溃后不会自动重启
- 无法监控运行状态

PM2 解决了这些问题：
- 后台运行
- 崩溃自动重启
- 进程监控
- 日志管理
- 负载均衡（集群模式）

### 5.2 常用命令

```bash
# 启动应用
pm2 start npm --name "myapp" -- start
pm2 start dist/main.js --name "myapp"

# 查看进程列表
pm2 list

# 查看详细日志
pm2 logs myapp

# 重启
pm2 restart myapp

# 停止
pm2 stop myapp

# 删除
pm2 delete myapp

# 监控面板
pm2 monit

# 开机自启配置
pm2 startup
pm2 save
```

### 5.3 配置文件

创建 `ecosystem.config.js`：

```javascript
module.exports = {
  apps: [
    {
      name: 'rbac-backend',
      script: 'dist/main.js',
      cwd: '/root/rbac-project',
      instances: 1,
      autorestart: true,
      watch: false,
      max_memory_restart: '500M',
      env: {
        NODE_ENV: 'production',
        PORT: 3000
      },
      env_production: {
        NODE_ENV: 'production',
        DB_HOST: 'localhost',
        DB_PORT: 3306,
        DB_USER: 'rbac_user',
        DB_PASSWORD: 'your_password',
        DB_NAME: 'rbac_db',
        JWT_SECRET: 'your_super_secret_key'
      },
      error_file: './logs/error.log',
      out_file: './logs/out.log',
      log_date_format: 'YYYY-MM-DD HH:mm:ss'
    }
  ]
};
```

**使用配置文件启动：**
```bash
pm2 start ecosystem.config.js --env production
```

### 5.4 集群模式（多核利用）

Node.js 单线程无法利用多核，PM2 集群模式自动负载均衡：

```bash
pm2 start dist/main.js -i max  # 最大实例数
pm2 start dist/main.js -i 4    # 4个实例
```

⚠️ 注意：集群模式下，内存共享数据（如 `global` 变量）不互通。

## 六、MySQL 数据库

### 6.1 数据库基础概念

- **数据库（Database）**：存储数据的仓库
- **表（Table）**：类似 Excel 表格，有行有列
- **列（Column）**：字段，如 id、name、age
- **行（Row）**：一条记录

### 6.2 常用操作

**登录 MySQL：**
```bash
mysql -u root -p
```

**基础 SQL：**
```sql
-- 查看所有数据库
SHOW DATABASES;

-- 创建数据库
CREATE DATABASE myapp;

-- 使用数据库
USE myapp;

-- 创建表
CREATE TABLE users (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(50) NOT NULL,
    email VARCHAR(100) UNIQUE,
    password VARCHAR(255) NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 查看表结构
DESC users;

-- 插入数据
INSERT INTO users (name, email, password) VALUES ('张三', 'zhangsan@example.com', '123456');

-- 查询数据
SELECT * FROM users;
SELECT name, email FROM users WHERE id = 1;

-- 更新数据
UPDATE users SET name = '李四' WHERE id = 1;

-- 删除数据
DELETE FROM users WHERE id = 1;
```

### 6.3 数据库备份与恢复

```bash
# 备份
mysqldump -u root -p myapp > backup.sql

# 恢复
mysql -u root -p myapp < backup.sql

# 备份所有数据库
mysqldump -u root -p --all-databases > all_backup.sql
```

### 6.4 远程连接配置

默认 MySQL 只允许本地连接，需要远程访问时：

```sql
-- 创建允许远程连接的用户
CREATE USER 'deploy'@'%' IDENTIFIED BY 'your_password';
GRANT ALL PRIVILEGES ON myapp.* TO 'deploy'@'%';
FLUSH PRIVILEGES;

-- 修改 MySQL 配置
vi /etc/mysql/mysql.conf.d/mysqld.cnf

# 找到 bind-address = 127.0.0.1
# 修改为 bind-address = 0.0.0.0

# 重启
systemctl restart mysql
```

## 七、SSL 证书配置

### 7.1 为什么需要 HTTPS？

- **安全**：加密传输，防止窃听
- **信任**：浏览器显示安全锁
- **SEO**：搜索引擎排名加分
- **要求**：微信小程序必需 HTTPS

### 7.2 免费证书 Let's Encrypt

**安装 Certbot：**
```bash
apt install certbot python3-certbot-nginx -y
```

**申请证书：**
```bash
certbot --nginx -d your-domain.com

# 多域名
certbot --nginx -d example.com -d www.example.com
```

**自动续期：**
Let's Encrypt 证书有效期90天，Certbot 自动续期。

```bash
# 测试续期
certbot renew --dry-run

# 查看定时任务
systemctl list-timers | grep certbot
```

### 7.3 手动申请证书（阿里云/腾讯云）

1. 云服务商控制台 → SSL证书 → 申请免费证书
2. 下载证书文件
3. 上传到服务器
4. 配置 Nginx

**Nginx 配置 SSL：**
```nginx
server {
    listen 443 ssl;
    server_name your-domain.com;

    ssl_certificate /etc/ssl/certs/your-cert.pem;
    ssl_certificate_key /etc/ssl/private/your-key.pem;

    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;

    location / {
        proxy_pass http://127.0.0.1:3000;
    }
}

# HTTP 重定向到 HTTPS
server {
    listen 80;
    server_name your-domain.com;
    return 301 https://$server_name$request_uri;
}
```

## 八、项目部署实战

### 8.1 项目结构说明

我们以 RBAC 权限管理系统为例：

```
rbac-project/
├── backend/              # 后端 (NestJS)
│   ├── src/
│   ├── dist/             # 编译输出目录
│   ├── package.json
│   └── ...
├── frontend/             # 前端 (React)
│   ├── src/
│   ├── dist/             # 编译输出目录
│   ├── package.json
│   └── ...
└── docs/
```

### 8.2 前端部署

**步骤1：打包前端**
```bash
cd /root/rbac-project/frontend

# 安装依赖
npm install

# 打包生产版本
npm run build
```

**步骤2：配置 Nginx**
```bash
vi /etc/nginx/sites-available/rbac
```

```nginx
server {
    listen 80;
    server_name 43.167.209.167;

    root /root/rbac-project/frontend/dist;
    index index.html;

    # 前端路由支持
    location / {
        try_files $uri $uri/ /index.html;
    }

    # API 代理
    location /api/ {
        rewrite ^/api/(.*) /$1 break;
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

**步骤3：启用配置**
```bash
ln -s /etc/nginx/sites-available/rbac /etc/nginx/sites-enabled/
nginx -t
systemctl reload nginx
```

**步骤4：带路径前缀的部署**
如果需要访问路径 `/rbac/`：

```nginx
server {
    listen 80;
    server_name 43.167.209.167;

    root /root/rbac-project/frontend/dist;

    # 访问 /rbac/ 时返回 index.html
    location /rbac/ {
        alias /root/rbac-project/frontend/dist/;
        try_files $uri $uri/ /rbac/index.html;
    }

    # API 代理
    location /rbac/api/ {
        rewrite ^/rbac/api/(.*) /$1 break;
        proxy_pass http://127.0.0.1:3000;
    }
```

### 9.1 域名是什么？

域名是网站的地址，如 `example.com`。

**域名结构：**
```
example.com
├── .com     顶级域名（TLD）
└── example  二级域名
```

### 9.3 DNS 解析配置

DNS 把域名指向服务器 IP。

**添加 DNS 记录：**

| 记录类型 | 主机记录 | 记录值 | 说明 |
|---------|---------|-------|-----|

| A | www | 43.167.209.167 | www 子域名 |
| A | api | 43.167.209.167 | API 子域名 |

## 十、安全防护

### 10.1 防火墙配置

**使用 UFW（Ubuntu 默认防火墙）：**
```bash
# 查看状态
ufw status

# 允许 SSH（重要！先开再启）
ufw allow 22/tcp

# 允许 HTTP/HTTPS
ufw allow 80/tcp
ufw allow 443/tcp

# 允许特定端口
ufw allow 3000/tcp

# 启用防火墙
ufw enable

# 查看规则
ufw status numbered
```

**使用云服务商安全组：**

在阿里云/腾讯云控制台 → 安全组 → 添加规则：

```
入站规则：
- 协议：TCP
- 端口：22
- 来源：0.0.0.0/0（你的IP更安全）

- 协议：TCP
- 端口：80,443
- 来源：0.0.0.0/0
```

### 10.2 SSH 安全

**修改 SSH 默认端口：**
```bash
vi /etc/ssh/sshd_config

# 找到 #Port 22
# 改为 Port 2222

# 重启 SSH
systemctl restart sshd
```

**禁止密码登录，使用密钥：**
```bash

# 修改以下配置

PubkeyAuthentication yes

systemctl restart sshd
```

```bash
netstat -tlnp | grep 3000
```

### 12.2 日志分析

**Nginx 错误日志分析：**
```bash

awk '{print $1}' /var/log/nginx/access.log | sort | uniq -c | sort -rn | head -10
```
